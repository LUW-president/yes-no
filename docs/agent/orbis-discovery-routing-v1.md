# ORBIS Discovery Routing v1

Purpose: align ORBIS and Discovery outputs with safe implementation sequencing.

## Flow
1. ORBIS Agent produces research-cycle insights.
2. Discovery Agent extracts repeatable signals and prototype candidates.
3. Strategic Intelligence ranks candidate leverage.
4. Architecture validates boundary fit.
5. Build may implement only approved deterministic prototype slices.

## Output contract
Each ORBIS/Discovery cycle should include:
- signal statement
- confidence level
- evidence source
- prototype candidate
- risk notes
- promotion recommendation (research-only / prototype-ready)

## Guardrail
No direct runtime implementation from ORBIS research artifacts without explicit promotion gate.
