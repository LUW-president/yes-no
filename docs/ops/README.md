# Ops Control Docs

## Purpose
This folder defines the durable control layer for autonomous/assisted execution inside the YES/NO repository.

## Role of `AGENT_DIRECTIVE.md`
Defines mission, non-negotiable principles, approval boundaries, autonomous permissions, and reporting format.

## Role of `PROJECT_STATE.md`
Provides the current phase snapshot, baseline state, active commands, test status, and next milestone.

## Role of `AGENT_LOOP.md`
Defines the repeatable execution loop agents must follow each cycle, including test + reporting discipline.
