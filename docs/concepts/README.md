# Concepts Folder

## Purpose of concept documentation
This folder captures exploratory concepts and architectural thought models that may inform future work.

Concept docs exist to clarify ideas early, reduce ambiguity, and preserve clean boundaries between current implementation and future exploration.

## Implemented features vs exploratory ideas
- Implemented features: reflected in runtime code, tests, and active runbooks/specs tied to shipped prototype behavior.
- Exploratory ideas: conceptual only, not shipped, not implied as current capability.

When in doubt, treat concept documents as non-shipping references unless explicitly promoted into an implementation phase.
