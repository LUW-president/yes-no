# ORBIS v0 (Concept)

## Concept overview
ORBIS is an exploratory spatial interaction concept centered on intentional global aiming.

**Status:** Concept stage only (exploratory).  
**Not implemented in the current repository.**

## Core interaction ("Aim anywhere on Earth")
A user points or aligns their device toward a chosen place on Earth and receives directional confirmation through a minimal interface.

The interaction prioritizes:
- focus
- orientation
- intentionality
- low interface clutter

## LOCK interaction moment
"LOCK" represents the moment where user aim and target alignment are accepted as intentional.

Conceptual properties:
- explicit user action
- minimal feedback
- no excessive animation
- clear state transition from aiming -> locked

## Snapshot links
A lock event may produce a shareable snapshot reference (conceptually), representing:
- selected target
- lock timestamp
- optional metadata envelope

Status: conceptual only in v0.

## Lock sessions
A lock session is a bounded interaction window:
- begin aiming
- stabilize direction
- lock target
- end session

Session framing avoids endless interaction loops.

## AR mode
Potential future mode:
- overlay directional guidance in camera view
- preserve minimal signal style

Status: exploratory concept, not implemented.

## Privacy principles
Concept-level privacy principles:
- explicit user consent before any share/export action
- minimal retained data by default
- no hidden location sharing
- user-visible state when lock/session data exists

## Example use cases
- directional ritual toward a meaningful location
- geospatial creative prompts
- educational/global orientation experiences
- private personal lock journaling

Scope note:
All use cases are conceptual and should not be interpreted as shipped product capabilities.
