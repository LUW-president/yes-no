# External Demo Pilot v0 — YES/NO

## purpose
Define a safe, controlled pilot framework for limited external YES/NO demos while preserving clarity, credibility, and product-scope discipline.

## pilot scope
- limited external sharing only
- controlled walkthroughs only
- no broad distribution
- no claims beyond prototype maturity

## intended audience
- selected design collaborators
- selected technical collaborators
- selected early partners
- selected early investors

## approved sharing posture
- limited external sharing only
- controlled sessions with explicit framing
- prototype maturity and limitations disclosed upfront

## demo format
- short guided walkthrough (scripted)
- deterministic demo path
- concise Q&A
- structured feedback capture immediately after demo

## materials to use
- investor brief
- product one-pager
- prototype package
- prototype walkthrough script
- prototype demo runbook
- internal sharing bundle
- share decision summary

## limitations that must be disclosed
- prototype only
- no production deployment
- gesture recognition is prototype-level
- no notification system yet
- roadmap features are not shipped

## what success looks like
- reviewer understands product thesis
- reviewer understands interaction model
- demo runs cleanly
- limitations are understood
- feedback is captured in structured form

## what must not happen
- no uncontrolled sharing
- no raw prototype sent without framing
- no overstatement of readiness
- no roadmap promises presented as shipped reality
