# Prototype Walkthrough Script v0 — YES/NO

## Purpose
This document is the spoken walkthrough script for presenting the YES/NO native mobile prototype.

Its role is to help present the product clearly, consistently, and with the right emphasis on simplicity and interaction design.

## Audience
- internal team
- design collaborators
- technical collaborators
- early partners
- early investors

## Opening Framing
“YES/NO is a radically simple AI interface. The system asks questions, and the user answers only YES or NO. Our goal is to replace prompts and interface complexity with guided binary decisions.”

## Step-by-Step Spoken Demo Flow
Use this exact live walkthrough path:

1. **Session starts**  
   “We’re starting a fresh session. Notice the interface is intentionally minimal from the first moment.”

2. **First question appears**  
   “The system presents one focused question—no prompt box, no menus, no branching UI noise.”

3. **Answer YES**  
   “I answer with a single binary decision: YES.”

4. **Next question appears**  
   “The next question arrives based on that decision. The flow stays one-question-at-a-time.”

5. **Answer YES**  
   “Again, no prompt writing—just a direct binary signal.”

6. **Next question appears**  
   “You can see the system progressively narrows intent through guided questioning.”

7. **Answer NO**  
   “Now I answer NO, which shifts the path deterministically.”

8. **Artifact proposal appears**  
   “The prototype now proposes an artifact generated from this guided decision path.”

9. **Artifact accepted**  
   “I accept the artifact using the same YES/NO contract.”

10. **Completion screen appears**  
    “The session reaches a clear completion state—simple, finite, and calm.”

## Key Product Messages
- “The questions are the product.”
- one question at a time
- binary interaction only
- black-screen minimalism
- calm pacing and intentional silence
- the prototype is proving the interaction model, not feature breadth

## Artifact Reveal Talking Points
- The system narrows intent through binary inputs.
- The artifact is a result of guided discovery.
- The UI remains minimal even at the moment of creation.

## Completion Talking Points
- The session has a clear end.
- The product should feel calm, not endless.
- The interaction can restart without introducing complexity.

## Closing Statement
“YES/NO demonstrates a new interaction model for AI: fewer controls, clearer decisions, and guided outcomes through binary flow. This matters because it removes prompt friction and makes intelligent interaction feel natural. The next milestone is native mobile UI prototype refinement and gesture-driven interaction.”

## Constraints
- documentation only
- reflect actual prototype behavior only
- concise and presentation-ready
- no speculative features beyond current roadmap
