# Prototype Walkthrough Script v0 — YES/NO

## purpose
Provide a concise, repeatable spoken script for presenting the YES/NO native mobile prototype live.

## audience
- internal team members
- early collaborators
- potential partners/investors

## opening framing
“YES/NO is a radically simple AI interface. Instead of writing prompts, the system asks one question at a time, and the user answers only YES or NO. The questions are the product.”

## step-by-step spoken demo flow
1. Start session and show first question.
   - “We begin with a single focused question on a minimal black screen.”
2. Answer YES.
   - “Each response is binary and deliberate—no typing, no prompt composition.”
3. Show next question and answer YES.
   - “The system guides the user through a structured decision path.”
4. Show next question and answer NO.
   - “Different YES/NO paths shape the outcome deterministically.”
5. Artifact proposal appears.
   - “At this point, the system proposes an artifact based on the decision flow.”
6. Accept artifact.
   - “The user confirms with the same binary interaction model.”
7. Completion screen appears.
   - “Session lifecycle completes cleanly without interface complexity.”

## key product messages
- one question at a time
- binary interaction only
- minimal interface, maximum clarity
- deterministic flow built on structured questioning

## artifact reveal talking points
- artifact appears as a direct output of the decision path
- flow remains calm and minimal at the reveal moment
- acceptance uses the same YES/NO contract

## completion talking points
- completion is explicit and lightweight
- no extra menus or navigation required
- user can restart and run another flow consistently

## closing statement
“YES/NO demonstrates a new interface layer for AI: fewer controls, clearer decisions, and guided outcomes—starting with a native mobile experience and expanding into a broader platform.”
