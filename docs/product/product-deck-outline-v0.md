# Product Deck Outline v0 — YES/NO

## product thesis
YES/NO is a radically simple AI product where users never write prompts—only answer one question at a time with YES or NO.

## problem
Current AI interfaces overload users with prompt complexity, feature noise, and cognitive overhead, reducing clarity and trust.

## interaction model
- one-question-at-a-time interface
- binary response contract (YES / NO)
- calm pacing, intentional silence, minimal visuals
- no prompt composition surface

## why now
- AI adoption is broad, but interface fatigue is rising
- users want speed with less cognitive burden
- mobile-first behavior favors simple decision loops over text-heavy input

## system architecture summary
- deterministic protocol + question engine + memory engine
- session orchestrator as transition authority
- bridge API for external clients
- mobile/web adapters for client integration
- reference apps validating interaction flow

## current baseline
- internal v0 baseline complete
- yesno command suite: run, harness, demo, status
- bridge + adapters + reference apps operational
- full baseline module test matrix passing

## visual direction
- black-screen aesthetic
- centered typography
- minimal/no decoration
- gesture-ready YES/NO interaction zones
- artifact and completion states with equal simplicity

## next milestone
Native mobile UI prototype demonstrating binary interaction, bridge integration, artifact flow, and timing/gesture-aligned UX.

## long-term platform vision
YES/NO evolves from a minimal interaction product into a multi-surface decision platform (mobile, web, spatial, voice-assisted) while preserving the same binary core and product philosophy.
