# Pilot Execution Checklist v0 — YES/NO

## purpose
Provide a practical checklist for running controlled external pilot demos safely, consistently, and with structured feedback capture.

## pilot session prerequisites
- limited external sharing posture approved
- presenter has reviewed current walkthrough + runbooks
- prototype baseline is stable in local demo environment
- feedback capture artifacts are prepared

## pre-demo setup checklist
- [ ] repo up to date
- [ ] prototype command verified
- [ ] bridge / run path verified if needed
- [ ] walkthrough script reviewed
- [ ] limitation disclosure ready
- [ ] feedback template ready

## approved materials checklist
- [ ] investor brief
- [ ] product one-pager
- [ ] prototype package
- [ ] prototype walkthrough script
- [ ] external demo runbook
- [ ] internal sharing bundle
- [ ] share decision summary

## disclosure checklist
- [ ] prototype only
- [ ] no production deployment
- [ ] gesture recognition is prototype-level
- [ ] roadmap items are not shipped
- [ ] demo is controlled and exploratory

## demo execution checklist
- [ ] start prototype cleanly
- [ ] follow walkthrough script
- [ ] avoid improvising roadmap promises
- [ ] keep scope tight
- [ ] note observed confusion points

## feedback capture checklist
- [ ] capture immediate reactions
- [ ] record quotes
- [ ] record confusion points
- [ ] classify signal as keep / investigate / ignore for now

## post-demo follow-up checklist
- [ ] complete feedback log template
- [ ] produce pilot session summary
- [ ] identify repeated signals vs one-off opinions
- [ ] propose only scoped refinements
- [ ] archive session notes in shared internal folder

## constraints
- documentation only
- practical and concise
- aligned with current limited external sharing posture
- no speculative features
- protect YES/NO simplicity
- avoid feature creep from single-session feedback
