# External Demo Runbook v0 — YES/NO

## purpose
Provide a practical, controlled operating runbook for limited external YES/NO prototype demos.

## prerequisites
- repository synced to approved baseline
- prototype runnable in local demo environment
- presenter has reviewed walkthrough script
- feedback protocol/template prepared

## approved audience
- selected design collaborators
- selected technical collaborators
- selected early partners
- selected early investors

## pre-demo checklist
- [ ] prototype runs cleanly
- [ ] commands verified
- [ ] walkthrough script reviewed
- [ ] limitation disclosure ready
- [ ] feedback capture template ready

## demo sequence
1. Set context and disclose prototype status.
2. Run guided deterministic demo flow.
3. Highlight interaction model and core thesis.
4. Confirm completion state and limitations.
5. Transition directly into structured feedback questions.

## disclosure script
Before demo, explicitly state:
- prototype only
- no production deployment
- gesture recognition is prototype-level
- roadmap features are not shipped
- demo is controlled and exploratory

## feedback capture step
- collect reactions immediately after demo
- record structured notes
- do not debate roadmap live
- do not improvise major feature promises

## post-demo follow-up
- classify feedback (keep / investigate / ignore for now)
- identify repeated patterns
- summarize actionable items without scope drift
- circulate concise internal recap

## constraints
- controlled sharing only
- no raw distribution beyond approved audience
- no overstatement of readiness
- keep narrative aligned to current baseline
